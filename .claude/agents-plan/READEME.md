# agents-plan

This directory stores execution plans generated by agent skills during the 6-step common workflow.

## Structure

Each feature or task gets its own subdirectory:

```
agents-plan/
  [feature-name]/
    plan-step-1.md
    plan-step-2.md
    plan-step-3.md
    ...
```

## How Plans Are Created

Plans are generated during **Step 02 (`02-plan-and-split-steps.md`)** of the common workflow, before any code is written. The agent splits the task into discrete plan steps and writes them here.

## Plan Reuse

Before starting any task, the agent checks this directory for an existing plan related to the current feature. If a matching plan is found, it is reused directly — no re-analysis needed. This saves tokens and avoids redundant work.

## Workflow Reference

| Step | File | Description |
|------|------|-------------|
| 01 | `01-analyze-qa.md` | Analyze requirements and confirm scope |
| 02 | `02-plan-and-split-steps.md` | Create this plan directory and write step files |
| 03 | `03-implement-plan-step-loop.md` | Implement each plan step |
| 04 | `04-test-loop.md` | Test and fix until green |
| 05 | `05-code-review-security-and-improvements.md` | Code review and security audit |
| 06 | `06-explain-changes-and-final-review.md` | Final summary and delivery |

Each step requires explicit user confirmation before the agent proceeds to the next.
